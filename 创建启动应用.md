# 创建博客 CMS 启动应用

## 方法一：使用 Automator（推荐）

### 步骤：

1. **打开 Automator**
   - 打开「启动台」> 搜索「Automator」
   - 或使用 Spotlight (⌘ + 空格) 搜索「Automator」

2. **创建新应用**
   - 点击「新建文稿」
   - 选择「应用程序」> 点击「选取」

3. **添加运行 Shell 脚本动作**
   - 在左侧搜索框输入「运行 Shell 脚本」
   - 将「运行 Shell 脚本」拖到右侧工作区

4. **配置脚本**
   - 在「Shell」下拉菜单选择：`/bin/bash`
   - 在脚本框中粘贴以下内容：
   ```bash
   /Users/whitney/Downloads/minimal-blog/start-blog-cms.sh
   ```

5. **保存应用**
   - 点击菜单栏「文件」>「存储」
   - 命名为：`博客 CMS`
   - 保存位置：桌面或应用程序文件夹
   - 文件格式：应用程序
   - 点击「存储」

6. **完成！**
   - 现在你可以双击「博客 CMS」图标来启动了
   - 第一次运行可能需要授权访问权限

---

## 方法二：创建可执行的 .command 文件（更简单）

### 步骤：

已经为你创建好了 `start-blog-cms.sh` 文件，你只需要：

1. **重命名文件**
   ```bash
   cp /Users/whitney/Downloads/minimal-blog/start-blog-cms.sh ~/Desktop/博客CMS.command
   chmod +x ~/Desktop/博客CMS.command
   ```

2. **双击运行**
   - 在桌面上找到「博客CMS.command」
   - 双击即可启动

---

## 自定义应用图标（可选）

1. 下载一个你喜欢的图标（PNG 或 ICNS 格式）
2. 右键点击「博客 CMS.app」> 选择「显示简介」
3. 将图标图片拖到简介窗口左上角的小图标上
4. 完成！

---

## 使用说明

启动后，系统会自动：
1. 检查并安装依赖（仅首次需要）
2. 在新的 Terminal 窗口启动开发服务器
3. 自动打开浏览器访问 `http://localhost:3000/admin`
4. 显示系统通知提醒启动成功

---

## 停止服务器

关闭 Terminal 窗口或按 `Ctrl + C` 即可停止服务器。
